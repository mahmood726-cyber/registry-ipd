# Downloadable pseudo-IPD bundle

The registry-wide reconstruction (`SCALE.md`) materialised as an **analysis-ready dataset** — the actual
deliverable of this project: reconstructed individual-patient survival data others can pool, with full
provenance. Generated by `node validate/export_bundle.js` (Titman-QP default).

## What's here

- **`dist/pseudo_ipd/registry_pseudo_ipd.csv`** — the full dataset: **886 exportable comparisons,
  259,369 pseudo-patient rows** stacked in tidy long format.
- **`realipd/pseudo_ipd_manifest.json`** — one row per comparison (NCT, condition, arm labels, N,
  events, reconstructed HR, registry HR, self-audit badge, method, anchors, source URL). Metadata only.
- **`dist/pseudo_ipd/SAMPLE_*.csv`** — six worked examples for quick format inspection.

## Format (the standard IPD-meta-analysis input)

```
nct,comparison,arm,time,status
NCT00878709,12,exp,8.3,1
NCT00878709,12,exp,11.0,0
NCT00878709,12,ctl,4.1,1
...
```

- `comparison` — integer id (a multi-arm trial contributes several); cross-reference the manifest.
- `arm` — `exp` (experimental) or `ctl` (comparator).
- `time` — event/censor time in the trial's native unit. `status` — `1` = event, `0` = censored.

## Use

Pool directly in any IPD survival meta-analysis (one-stage frailty Cox, or two-stage per-trial HR →
random-effects). The reconstruction preserves the pooled effect well (see `VALIDATION.md` →
*IPD meta-analysis fidelity*: pooled HR within ~6% of true IPD, with a small **conservative**
heterogeneity inflation). Filter the manifest to `badge ∈ {gold, silver}` and `method = qp` for the
highest-fidelity subset (the QP fires where a registry event count is posted).

## Honest notes

- The pseudo-IPD is **reconstructed (synthetic)** — freely shareable, **not real patient data**.
- Per-trial fidelity is bounded by the posted anchor density and whether an event count is available
  (the binding limits this project documents). Read each comparison with its badge; treat the HR as a
  triangulation input, the RMST/median as tighter.
- The dataset regenerates exactly from the harvested cohort: `node validate/export_bundle.js`.
