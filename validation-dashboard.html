Registry-IPD — Validation Dashboard

Reconstructing pseudo individual-patient survival data from ClinicalTrials.gov / AACT summary data only, validated up to true patient-level RCT data. Fully offline. See VALIDATION.md / METHODS.md in the repo for full detail and caveats.

Headline verdict (honest)

For adequately-sized trials, RMST and median reconstruct to ~2–6% and the HR to ~12% (median); the 95% credible interval covers the true HR 23/24 (96%) on real data. HR is the hard estimand; very small trials don't reconstruct; needs ≥5–6 posted KM timepoints. Not a substitute for true IPD — a triangulation tool with honest uncertainty.

AACT coverage — what is actually reconstructable

Full 2026-06-01 snapshot: 76,067 trials with posted results, but only 288–~605 (0.4–0.8%) post a reconstructable KM curve and 0 post a structured number-at-risk row. Of curve-posters, only ~34% reach the ≥6 timepoints needed for reliable reconstruction (green region). Bars: trials by posted KM-timepoint count (broad detection). Source: census_full_aact.py.

Head-to-head — registry-structured vs figure-digitization

Same true curve, two posted forms, both reconstructed vs truth (25 real datasets). Median |log HR| error. At equal density it's a near-tie (pixel noise costs little); a dense digitized figure (K=25) beats few exact registry anchors (K=8) — the registry path is anchor-density-limited, not noise-limited. headtohead.js.

Digitization-noise sensitivity

Sweeping the digitization noise σ. Registry-exact is the flat noise-free reference; equal-density digitized crosses above it as noise grows, but K=25 digitized stays below across the whole 0–5 pp range. The conclusion is not an artefact of the chosen noise level. noise_sweep.js.

Real cancer cohorts (TCGA) — why the event-count tier matters

12 real TCGA overall-survival cohorts (open cBioPortal API, fetch_cbioportal.js) split by late vs early stage — a strong contrast (true HR 1.6–7.6), heavily censored. Curve-only underestimates these large HRs (median fold 1.56: the early-stage arm is mostly censored, flattening the separation). The censoring-informed tier — using the registry total-event count — recovers them (median fold 1.20; e.g. colorectal 3.11 true → 1.70 curve-only → 3.11; kidney-papillary fold 2.41 → 1.14). Bars = HR fold-error per cohort; lower is better. Advanced curve-only estimators (max-entropy ensemble, Wasserstein barycenter) can't close this gap — the event count is the missing information (advanced_estimators.js), which is the case for POLICY.md.
